I-1 · Integration Epic
Product & Category Sync — RamBase → Struct via Harmony

Sync product master data and category tree from RamBase (ERP) into Struct (PIM) via Mosaik Harmony, enabling Pretec to enrich products for the storefront.

Status:  Written Type: Integration (INT) Direction: One-way — RamBase → Harmony → Struct Spec date: 2026-06-19
Data Flow
RamBase (product master / category tree) │ ▼ Mosaik Harmony — Product & Category Sync │ ▼ Struct (PIM) — Pretec enrichment │ ▼ [downstream → I-2 Catalog Publish] Storefront Catalog
User Stories
  • 1
    As Pretec content team, when a new product is created in RamBase, I want it to appear in Struct automatically so I can start enriching it without manual data entry.
  • 2
    As Pretec content team, when ERP fields are updated in RamBase, I want my Struct enrichment (descriptions, images, SEO) to be preserved — only RamBase-owned fields should change.
  • 3
    As Pretec content team, when a product is deactivated in RamBase, I want a predictable, agreed behavior in Struct (hidden / unpublished / shown as unavailable).
  • 4
    As Pretec content team, I want category assignments to be available in Struct so I can organise the storefront catalogue by browsing category.
  • 5
    As Geta operations, when a product fails to sync I want a clear error log with product ID and reason so I can investigate and retry without manual cleanup.
Implementation Tasks
  • Confirm which RamBase product endpoint(s) and fields are exported by Harmony
  • Confirm which RamBase category / classification structure becomes the storefront category tree
  • Agree field ownership mapping (RamBase-owned vs. Struct-owned) field by field
  • Confirm which RamBase field marks a product as web/catalog eligible
  • Confirm whether product name in ERP is used as display name or Struct overrides it
  • Configure Harmony product & category sync mapping and transformations
  • Configure Struct product model and target attributes required for listing, filtering, and PDP
  • Decide and configure sync frequency (scheduled hourly/nightly vs. near-real-time)
  • Agree and implement product deactivation behavior in Struct and downstream catalog
  • Set up error reporting for failed product syncs with retry capability
  • Define and apply minimum product data quality rules
  • Confirm initial product volume and expected daily change volume for sizing
  • Confirm operational owner for sync failure follow-up (Geta vs. Pretec)
  • Confirm whether Struct may create storefront-only categories for merchandising
  • Execute initial full product & category load

Success Criteria
  • ✓
    Products created or updated in RamBase are created or updated in Struct by Harmony.
  • ✓
    RamBase-owned fields stay controlled by RamBase; Struct enrichment is never overwritten.
  • ✓
    Category assignments from RamBase are available in Struct for catalogue organisation.
  • ✓
    Product sync failures are visible, actionable, and retryable.
  • ✓
    Enriched Struct product data can be consumed by the downstream Catalog Publish (I-2).
