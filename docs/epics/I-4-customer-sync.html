I-4 · Integration Epic
Customer Sync — RamBase ↔ Mosaik via Harmony

Two-way customer sync: existing company/contact data flows RamBase → Mosaik (inbound, scheduled). On new-company approval, Mosaik → RamBase creates the company and returns the RamBase ID (write-back).

Status:  Partial — consolidating Type: Integration (INT) Direction: Bi-directional (inbound scheduled + outbound on approval)
Data Flow
Direction 1 — Inbound (existing customers)
RamBase (existing companies + contact persons) │ ▼ Harmony — Customer Sync (scheduled) │ ▼ Mosaik Storefront User DB
Direction 2 — Write-back (new company application)
Storefront (new company registration form) │ ▼ Maestro — approval by Pretec Sales │ ▼ Harmony — Create Customer (on approval) │ ▼ RamBase (new company created) │ ▼ RamBase unique ID returned → stored in Mosaik account
User Stories
  • 1
    As an existing Pretec customer, when I log into the storefront for the first time my company and contact-person data from RamBase is already populated in my account.
  • 2
    As a new company applicant, once Pretec Sales approves my registration, my company is created in RamBase and I receive an invitation to activate my account linked to the correct RamBase customer ID.
  • 3
    As Pretec Sales, when I approve a new company application in Maestro, the company is created in RamBase automatically (with the extra information I fill in) so I don't have to create it twice.
  • 4
    As Pretec Sales, when an application names a company that already exists in RamBase, the system links the application to the existing company rather than creating a duplicate.
  • 5
    As Geta operations, if a customer sync or company-creation fails I want a visible alert with retry capability so no customer is permanently stuck.
Implementation Tasks

  • Define "create-company" payload — exact RamBase fields set on approval, including the extra information Pretec Sales fills in
  • Confirm how the returned RamBase unique ID is stored and linked to the Mosaik account
  • Define delta / incremental strategy for inbound sync (ties to C-1 Harmony config)
  • Confirm whether the approving user also becomes a RamBase contact person on the new company
  • Define de-duplication rule when application names a company that already exists in RamBase
  • Define conflict handling — RamBase as source of truth for existing company master data
  • Confirm customer deactivation handling (account disabled in storefront, GDPR/security implications)
  • Implement and test error handling for create-company failure mid-approval (partial failure recovery)

  • Configure Harmony inbound customer sync and validate field mapping with Pretec
