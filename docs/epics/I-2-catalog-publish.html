I-2 · Integration Epic
Catalog Publish — Struct → Storefront Catalog via Harmony

Publish enriched product and category data from Struct (PIM) to the Mosaik Storefront Catalog via Harmony Catalog Sync, making the catalogue available for browsing and search.

Status:  Stub — to write Type: Integration (INT) Direction: One-way — Struct → Harmony → Storefront Catalog Priority: #1 gap (completes the product flow)
Data Flow
[I-1 upstream] RamBase → Harmony → Struct (enriched PIM) │ ▼ Mosaik Harmony — Catalog Sync │ ▼ Mosaik Storefront Catalog (listing, PDP, navigation)
User Stories
  • 1
    As a storefront shopper, I want enriched product data (name, images, descriptions, attributes) from Struct to appear in product listing and detail pages so I can browse the full catalogue.
  • 2
    As a storefront shopper, I want product categories from RamBase (enriched in Struct) to appear as navigation so I can browse by category.
  • 3
    As Pretec content team, when I publish a product change in Struct I want it to appear in the storefront within the agreed sync window.
  • 4
    As Pretec content team, when I unpublish or deactivate a product, I want predictable storefront behavior (hidden / shown as unavailable) per agreed rules.
  • 5
    As Geta operations, when a catalog publish fails I want a visible error so I can retry without stale products appearing in the storefront.
Implementation Tasks
  • Define field mapping: Struct product fields → Storefront Catalog fields (field by field)
  • Define publish eligibility rules — what makes a product/category "ready to publish"
  • Confirm asset handling — how images and documents are transferred from Struct to the catalog/CDN
  • Define category tree shape in the Storefront Catalog and how it derives from Struct/RamBase
  • Confirm variant and product relationship handling in the catalog
  • Define unpublish / deactivation behavior (hide vs. unpublish vs. show as unavailable)
  • Configure Harmony Catalog Sync mapping and transformations
  • Set up initial full catalog load and validate output in Storefront
  • Configure sync frequency and delta load strategy
  • Define data-quality gates that block publishing of incomplete product records
  • Set up error handling and alerting for catalog publish failures
  • Confirm which catalog fields feed Search & Filtering (F-2) — coordinate with search team
Open Decisions

  • ⚠

  • ⚠
    Unpublish/deactivation behavior — hide from listing, show as unavailable, or full unpublish?
  • ⚠
    Which Struct fields (attributes, facets) are required for storefront listing and filtering (F-2)?
  • ⚠
    Sync frequency — how quickly must Struct changes appear in the storefront?
