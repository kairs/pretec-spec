I-3 · Integration Epic
Inventory Sync — RamBase → Storefront Catalog via Harmony

Synchronise stock and availability data from RamBase to the Mosaik Storefront Catalog via Harmony Inventory Sync, enabling the storefront to display product availability to shoppers.

Status:  Stub — to write Type: Integration (INT) Direction: One-way — RamBase → Harmony → Storefront Catalog Phase 1 assumption: Synced (not live)
Data Flow
RamBase (stock / availability records) │ ▼ Mosaik Harmony — Inventory Sync │ ▼ Storefront Catalog (availability display) │ ▼ Storefront renders availability bands to shoppers

Note: Price is fetched live from RamBase (not synced). Inventory is synced/cached — acceptable staleness window to be agreed.

User Stories
  • 1
    As a storefront shopper, I want to see product availability (in stock / low stock / out of stock) on listing and product detail pages so I know what I can order.
  • 2
    As a storefront shopper, I want availability to reflect recent stock changes within the agreed freshness window so I don't order products that are out of stock.
  • 3
    As Pretec, I want to decide whether exact quantities are shown or just stock-status bands (in/low/out) to align with business policy.
  • 4
    As Geta operations, when an inventory sync fails I want a visible alert so the team can investigate before stale availability misleads shoppers.
Implementation Tasks
  • Confirm which RamBase stock figure(s) to use: on-hand, available-to-promise, per warehouse, or combined
  • Agree availability display model — exact quantity vs. in-stock / low-stock / out-of-stock bands
  • Decide whether availability is shown for anonymous users or logged-in only
  • Confirm multi-warehouse / location handling scope for Phase 1
  • Set acceptable staleness window (e.g. 15 min, hourly) and communicate it on the storefront if needed
  • Configure Harmony Inventory Sync — source fields, mapping, frequency
  • Configure storefront catalog to receive and render inventory data from Harmony
  • Define behavior when sync is delayed or fails (show last known, show "contact us", hide availability)
  • Set up error alerting for inventory sync failures
  • Execute initial inventory load and validate display in storefront
